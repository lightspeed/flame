# Button

The most essential element of a user interface.

## Types, styles and sizes

Each buttons have multiple set of types to denote the user a visual
intent of an action associated to a button button. To change the color, simply use the `variant` prop.

Adding a `fill` prop will fill the color of the button. Please note that in the future, this will be the default option.

## Forcing state and behaviour

Button can also have forced state and behaviour. This is particularly handy when dealing
with dropdowns, where you would want to have the button in a "active" state while the
popover component is being displayed.

To do so, you may leverage the `forcedState` prop to set the state of the buttons.

## Disabling a button

Unsurprisingly, you can disable a button by passing in the `disabled` prop.

<Button
  disabled
  onClick={() => {
    alert('if you see me, something went wrong');
  }}
>
  Disabled Button
</Button>

## Loading button

A button can be set to a loading state by simply setting the `loading` prop to true.

When the button is set to loading, it is automatically disabled.

<Button
  loading
  onClick={() => {
    alert('if you see me, something went wrong');
  }}
>
  Default
</Button>

## Button as a link

The button will automatically be converted into an `<a>` tag if a `href` is specified.

<Button href="www.google.com">Visit Google</Button>

## Icons within a button

Adding an Icon to a button will cause it to automatically position the Icon to an optimal position.

We highly recommend to never add more than 1 icon to a button.

<Button>
  <IconApple /> This is an apple
</Button>
